# PDF 自动书签生成工具

自动识别 PDF 目录页内容，为 PDF 文件生成章节目录书签。（目前仅有 win 版本， mac 版本正在开发中）

## 功能特性

- **自动识别目录** — 提取 PDF 目录页文字，结合 LLM 解析章节标题与页码
- **视觉识别模式** — 将目录页渲染为图片，调用视觉大模型（需用户手动配置api）精准识别
- **CLI + Web 双界面** — 命令行版和浏览器版，适应不同使用场景
- **页码偏移自动计算** — 目录中相对页码 → PDF 实际页码自动映射
- **缺页检测** — 自动验证最后一个章节所在页码是否匹配，防止缺页/多余页
- **罗马数字过滤** — 自动跳过前言等页码为罗马数字的章节
- **大模型测试** — web版提供大模型测试功能，帮助用户排查错误原因
- **多引擎回退** — PyMuPDF → pdfplumber → Edge 浏览器 → LLM 四级回退

## 快速开始

### 1. 下载文件并安装依赖
点击文件列表右上方的绿色 Code 按钮，将下载好的压缩包解压并移动至你想要的位置，将文件夹中的'config.example.json'文件重命名为'config.json'。接下来在命令行窗口中(搜索框中输入 cmd 即可找到)输入如下代码

```bash
pip install -r requirements.txt
```

### 2. 配置 API
此程序要求用户提前自备好 API-Key 。有两种方式让程序能成功调用大模型：

2.1 编辑 `config.json`：

```json
{
    "llm_base_url": "https://api.deepseek.com/v1",
    "llm_api_key": "your-api-key-here",
    "llm_model": "deepseek-chat",
    "llm_vision_model": "",
    "vision_base_url": "",
    "vision_api_key": ""
}
```

- `llm_*` — 文本模型配置，用于文字识别回退
- `vision_*` — 视觉模型配置（可选），用于图片识别模式。留空则复用文本模型配置

2.2 跳过“配置 API ”这一步，直接进入“运行”步骤，初次运行时按照窗口提示一步步操作即可

### 3. 运行

**Web 界面（推荐）：**

```bash
python app_web.py
# 或双击 run_web.bat
```

浏览器自动打开 `http://127.0.0.1:5000`，上传 PDF → 输入页码信息 → 选择识别方式（强烈建议直接选择图片模式） → 点击"开始生成书签"。程序运行完成时即可下载写好书签的 PDF 文件到浏览器下载文件的默认位置。若程序运行时出错或想要为下一个 PDF 文件生成书签，点击“重置进程”按钮即可。

**命令行CLI界面：**

```bash
python pdf_bookmarker.py
# 或双击 run.bat
```

大致流程与 web 界面的相同，且选择识别模式时同样强烈建议选择“3-大模型识别”，主要区别在于CLI界面中暂无大模型测试功能，第一次使用时需在第一个弹窗中选择“否”来配置 API

## 使用说明

### 输入参数

| 参数 | 说明 |
|------|------|
| 目录开始页 | 目录第1页的 PDF 页码 |
| 目录结束页 | 目录最后1页的 PDF 页码 |
| 正文偏移量 | 正文第1页的 PDF 页码 - 1 |

### 计算公式

```
章节实际页码 = 正文偏移量 + 目录中章节相对页码
```

### 输出

在原始 PDF 同目录下生成 `副本_原文件名.pdf`，包含完整的目录书签。此程序还在改进中，建议新文件生成后用户手动点击查看新文件内容，对书签进行手动调整。

## 大模型选择
双击run_web.bat进入 web 界面后根据提示选择文本、视觉模型，也可以选择全模态模型（同时支持文本、图片识别）

注：质量尚可的一般视觉大模型都能较准确地识别 PDF 文件中的目录页信息，没必要选择价格偏高的高端大模型

## 常见问题

**为什么程序运行时弹窗提示我文件可能有遗漏/多余页？**

这是程序自带的功能，具体实现方式为识别目录页中最后一个章节的页码信息，再跳转至该页识别是否符合章节内容，若不符就说明 PDF 文件页码编排可能有错。偶尔会出现程序出错的情况，此时推荐用户手动检查一遍 PDF 文件，若确认无误则可在弹窗中点击“是”让程序继续生成书签。

**为什么目录识别不完整？**

可能原因：
1. 建议使用视觉模式（图片识别），文本模式依赖 PDF 文字提取质量，部分 PDF 文字层为乱码
2. 程序还有待提高。事实上，现有的提高目录识别质量的方法会导致 token 消耗量大幅增长，正在探索更节省的方案

**为什么某些章节的书签页码有偏差？**

可能原因：
1.  PDF 目录页印刷的页码本身不准确
2. 部分 PDF 电子书会删除一些原有的空白页，比如两章之间常见的空白页
3. 正文偏移量输入有误

程序会自动验证最后一个章节的页码准确性。

**支持哪些文件？**

目前仅支持 PDF 文件，包括文字型 PDF（可选择自动模式，更推荐直接选择视觉模式）和扫描版 PDF（需使用视觉模式）。适用 DOC 等格式文件的版本敬请期待。
